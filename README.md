# TaskFlow

A full-stack task management application with user authentication, task tracking, and daily streaks. Built with a FastAPI backend and a React frontend, backed by PostgreSQL.

Users register and log in, manage their own tasks, mark them complete each day, and build streaks as they stay consistent — all behind secure, token-based authentication.

## Highlights

- **Full-stack** — FastAPI REST API + React single-page frontend, talking over HTTP with JWT auth
- **Secure authentication** — user registration and login with hashed passwords and JWT access tokens; protected routes that require a valid token
- **Multi-user** — every task belongs to a user; each account only sees and manages its own data
- **Streak tracking** — mark tasks complete daily and watch consecutive-day streaks build
- **Clean, responsive UI** — dark-themed React interface with a live task dashboard, search, and a user profile page

## Tech Stack

**Backend**
- FastAPI — REST API framework
- PostgreSQL — relational database
- SQLAlchemy — ORM for models and queries
- Pydantic — request/response validation
- JWT + OAuth2 password flow — authentication
- Passlib / bcrypt — password hashing
- Uvicorn — ASGI server

**Frontend**
- React — component-based UI
- React Router — client-side routing
- Axios — HTTP client for the API
- React Context — shared state (search) across components
- Custom CSS — dark-themed, responsive design

## Features

- Register a new account and log in securely
- JWT-based sessions with protected API routes
- Create, edit, and delete tasks
- Mark tasks complete for the day and track daily streaks
- Dashboard showing total tasks, percent completed today, and current streak
- Search tasks from the navigation bar
- View a personal profile page with account details and logout

## API Overview

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/auth/register` | Register a new user |
| `POST` | `/auth/login` | Log in and receive a JWT access token |
| `GET` | `/users/me` | Get the currently authenticated user |
| `GET` | `/tasks/` | List the current user's tasks |
| `POST` | `/tasks/` | Create a new task |
| `GET` | `/tasks/{id}` | Get a single task |
| `PUT` | `/tasks/{id}` | Update a task |
| `DELETE` | `/tasks/{id}` | Delete a task |
| `POST` | `/tasks/{id}/complete` | Mark a task complete for today |
| `GET` | `/tasks/{id}/streak` | Get a task's current streak |

Full interactive API documentation is available at `/docs` when the backend is running (auto-generated by FastAPI).

## Getting Started

### Prerequisites

- Python 3.10+
- Node.js 18+
- PostgreSQL installed and running

### Backend Setup

1. From the project root, create and activate a virtual environment:
   ```bash
   python -m venv .venv
   # Windows
   .venv\Scripts\activate
   # macOS / Linux
   source .venv/bin/activate
   ```

2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. Create a PostgreSQL database named `taskflow`.

4. Create a `.env` file in the project root:
   ```
   DATABASE_URL=postgresql://postgres:YOUR_PASSWORD@localhost:5432/taskflow
   SECRET_KEY=your_jwt_secret_key
   ```

5. Run the backend:
   ```bash
   uvicorn app.main:app --reload
   ```
   The API runs at `http://localhost:8000` (interactive docs at `/docs`).

### Frontend Setup

1. Move into the frontend folder and install dependencies:
   ```bash
   cd frontend
   npm install
   ```

2. Start the development server:
   ```bash
   npm start
   ```
   The app runs at `http://localhost:3000`.

> Make sure the backend is running before using the frontend, since the UI makes authenticated requests to the API.
